# Аудит Логического Дерева L1 ➔ L4 и Цепочек Действий Бота

На основе предоставленных фотоматериалов, а также анализа исходного кода `AdvisorEngine.kt` и `RobotPlayer.kt`, был проведен фундаментальный аудит вычислительных цепочек от захвата (Origin) до выполнения конечного решения (Final Action).

## 1. Структура зон и расчетов

### **ЗОНА 1: Clean Math (Уровни L1 и L2)**
* **Уровень L1 (Bare GTO Math):** 
  Функция `computeRecommendation` складывает базовое математическое ожидание (BaseL1Score) из 4-х источников:
  1. `s1` — Монте-Карло симуляция на чистую победу.
  2. `s2` — Группы Склански-Малмута (Сила структуры префлоп/постфлоп).
  3. `s3` — Чен Рекордс (Математическая формула).
  4. `s4` — Разрыв потенциалов банка (Gap/Margin) к Pot Odds.
  * *Точка слияния:* `baseL1Score = (s1 * 0.35) + (s2 * 0.25) + (s3 * 0.20) + (s4 * 0.20)`
  * *Финальное решение L1:* Сравнивает победу с Pot Odds, и выдает механическое решение Call/Fold/Raise.

* **Уровень L2 (Оверлеи Персональной Статистики):**
  Берет `baseL1Score` как фундамент. На уровне `computeRecommendationL2`:
  Бежит по каждому активному оппоненту из числа тех, что есть за столом, суммируя поправки:
  1. `deltaSD` (Разница между WSD и WTSD, оценка честности оппонента).
  2. `preflopFocus` (Отклонение PFR к VPIP).
  3. Общая лузовость (VPIP).
  * *Точка слияния L2:* Вносит позицию за столом и стадию турнира/CM `l2Score = baseL1Score + netOpponentAdj + positionAdj + stageAdj + stackAdj`.
  * *Система отказоустойчивости:* Если игрок неизвестен, бот применяет средние значения (`avgTableVpip`, `avgTableWsd`), не давая логике "улететь в космос".

### **ЗОНА 2: Predictive / Synthesis Area (Уровень L3)**
Разветвление решений на 3 магистрали для поиска лучшего математического эксплойта в `computeRecommendationAdvanced`.
Берется фундамент из L1. Далее, применяется профайлинг оппонентов:
* **Генерация Архетипов:** "Акула", "Гиппопотам", "Гепард", "Обезьяна", "Хамелеон".
* **Разделение на Pathways (сценарии):**
  * `L2.5` - Пассивный путь (Сбор блефов прокаткой).
  * `L3.0` - Прямое Велью-Извлечение из сильной комбинации.
  * `L3.5` - Агрессивная ветка (Блеф за счет фолд-уязвимостей оппонентов).
* *Точка сжатия (Convergence Point):* Сравниваются 3 ветки `evL3_5 > evL3_0` и т.д. Бот выбирает **одну путь с наибольшим весом (bestBranch)** — `FINAL SCORE`.
* *Override-концы (Shortstack-push):* При M-Ratio < 12 на префлопе, любые пограничные решения `CALL` принудительно переписываются логикой либо в `ALL-IN`, либо в `FOLD`.

## 2. Разбор инцидента: "Почему он сфолдил, когда выбрал Колл?"

Исследовав логику в `RobotPlayer.kt` и UI экрана (на основании вашего скриншота, где бот нажал Fold против ставки All-In), можно проследить четкий путь:

1. Оппонент пошел "All-in".
2. OCR (захватчик) не успел распознать флоп (в HUD написано `Борд: --`), из-за чего модуль `AdvisorEngine` классифицировал стадию как **Preflop**. На префлопе Q4s - слабая рука (Группа Склански = 8).
3. Однако короткий размер активного стека заставил L1, L2 и L3 выдать первичное действие **CALL**, так как шансы банка посчитались выгодно (`pot odds` против маленького пуша). В HUD отображается: `L3 КОЛЛ (53%)`.
4. В самом конце функции `computeRecommendationAdvanced` есть **экстренный предохранитель короткого стека (Short-Stack Exploit Override)**:
```kotlin
if (mRatio < 12f && isPreflop) {
    if (sklanskyGroup <= 3 || finalScore > 0.58f) {
        finalAction = "ALL-IN"
    } else if (finalAction == "CALL") {
        finalAction = "FOLD" // ПРИНУДИТЕЛЬНЫЙ ПЕРЕХВАТ!
        explanation = "Пас укороченного стека префлоп"
    }
}
```
* **ИТОГ:** Из-за того, что бот посчитал, что это Префлоп турнир (поскольку не нашел карт на борде) и эффективный стек меньше 12 ББ (короткий стек), он применил пуш-фолд стратегию. Q4 не подходит под алл-ин префлоп (группа 8 > 3), поэтому бот перехватил решение "Колл" и принудительно изменил его на **FOLD**, чтобы не коллировать олл-ин с мусором. 

На скриншоте в HUD фраза осталась «Колл», но робот-кликер уже получил перехваченный экстренный FOLD.

## 3. Резюме
Архитектура совпадает 1 в 1 с вашими записями (рукописными набросками) из фотографий:
* **L1 и L2** следуют прямым линиям (Clean Math).
* **L3** разделяется на три вектора (2.5, 3.0, 3.5), суммируется и приводит к одному знаменателю (Convergence into Action).
* Генерируется **final** решение, которое передается на исполнение слоя L5 (RobotPlayer).
