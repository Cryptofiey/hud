# 🧪 M.A.R.S. (Monte-Carlo Adaptive Routines Simulator)
**Блок Симуляции, Проверок и Оптимизации Стратегий**

Этот изолированный модуль-песочница предназначен для симуляции игры, калибровки параметров ядра (уровни L1-L4) и проверки жизнеспособности стратегий бота. **Этот код не попадает в итоговый APK-файл Android-приложения**, он живет исключительно здесь в репозитории для наших математических тестов и анализов.

## 🛠 Доступные команды для ИИ-Ассистента

Вы можете отдавать мне команды в свободной форме прямо в чате. Я запущу необходимые скрипты, проанализирую выдачу и сообщу вам итоги. Механизм симулирует от сотен до тысяч раздач, слегка подкручивая коэффициенты (маленькими шагами), пока результат алгоритма не приблизится к максимуму профита.

### Команда 1: Запустить базовый прогон (Тест)
👉 *"Сделай прогон на 1000 раздач"*
В фоне выполняется: `npx -y zx scripts/optimizer/simulate.mjs --hands=1000 --opponents=reg:3`
*(Эта команда проверит, какую прибыль приносят ТЕКУЩИЕ настройки из `AdvisorEngine.kt`)*

### Команда 2: Оптимизировать конкретный параметр (Подбор)
👉 *"Сделай калибровку L2 параметров (например, potOddsMargin или shortStackThreshold) на 2000 раздач"*
В фоне выполняется: `npx -y zx scripts/optimizer/optimize.mjs --target=potOddsMargin --hands=2000`
*(Ассистент будет запускать симуляции в цикле небольшими шагами в пределах заданных диапазонов и найдет "золотую середину")*

### Команда 3: Экспорт (Перенос логики)
👉 *"Примени найденные оптимальные значения в основной код бота"*
*(Я возьму лучшие параметры, которые мы нашли в симуляторе, и аккуратно впишу их в `AdvisorEngine.kt` и другие компоненты бота)*

## 📂 Архитектура
- **`simulate.mjs`** — базовый движок симуляции. Роллит случайные ситуации (стеки, шансы, силу руки) или полные борды, имитируя игру нескольких оппонентов с разными скиллами и типажами.
- **`bot_logic.mjs`** — точный JS-порт ключевой математики из `AdvisorEngine.kt` с экспортированными "регулировочными вентилями". Это позволяет быстро тестировать изменения логики без перекомпиляции Kotlin-кода.
- **`optimize.mjs`** — движок сканирования диапазонов. Делает микро-шаги (например, изменяет маржу на 0.01) и строит график оптимальных решений (сводит математику к ожидаемому профиту).
