# ПОЯСНЕНИЯ К ФАЙЛАМ ПОКЕРНОГО HUD
## Что каждый файл делает и как они работают вместе

### 1. OverlayService.java
**Назначение:** Управление захватом экрана и отображением HUD-наложения

**Как работает:**
- Инициализирует MediaProjection для захвата содержимого экрана устройства
- Создает виртуальный display для непрерывного захвата кадров
- Обрабатывает изменения ориентации экрана
- Управляет жизненным циклом сервиса HUD (запуск, остановка, передний план)
- Создает и управляет UI элементами overlay (карты, чарты, информационные панели)
- Связывает систему захвата экрана с обработкой изображений через ChangeFinder

**Ключевые методы:**
- `startScreenCapture()` - начинает захват экрана
- `initVirtualDisplay()` - создает виртуальный display для захвата
- `destroyMediaProjection()` - очищает ресурсы захвата
- `addOverlay()` - добавляет UI элементы на экран
- `FindingsCallback` - обрабатывает результаты распознавания и обновляет HUD

### 2. Recognition.java
**Назначение:** Распознавание игральных карт из обработанных изображений

**Как работает:**
- Принимает сырые данные изображения и распознает символы карт (масть и ранг)
- Использует эвристические методы для определения наилучшего соответствия
- Поддерживает временную фильтрацию для уменьшения ложных срабатываний
- Предоставляет геометрические данные (расположение, размеры, отступы)
- Конвертирует распознанные данные в объекты Card для покерной логики

**Ключевые поля:**
- `title` - распознанная карта (например, "Ah" для туза червей)
- `confidence` - уровень уверенности в распознавании (0.0-1.0)
- `location` - прямоугольник с координатами карты на экране
- `allScores` - массив оценок для всех возможных рангов и мастей
- `timesInspected` - сколько раз эта карта проверялась для временной фильтрации

**Ключевые методы:**
- `toCard()` - конвертирует строку распознавания в объект Card
- `update()` - обновляет данные на основе новых наблюдений
- `compareTo()` - сравнивает две распознанные карты по сходству
- `getLocation()`, `getConfidence()` - геттеры для доступа к данным

### 3. PixelDetector.java
**Назначение:** Низкоуровневая обработка изображений для обнаружения карт

**Как работает:**
- Использует RenderScript для GPU-ускоренной обработки изображений
- Выполняет шаблонное сравнение captured изображений с эталонными шаблонами карт
- Выполняет вычитание фона для изолирования карт от стола
- Поддерживает обнаружение множества карт одновременно
- Возвращает статистику соответствия для фона и карт

**Ключевые компоненты:**
- RenderScript контекст и выделения памяти для обработки
- ScriptC_imagematcher и ScriptC_pixeldetector - скомпилированные скрипты RenderScript
- Allocation объекты для хранения промежуточных результатов
- MatchingData класс для возврата результатов сопоставления

**Ключевые методы:**
- `updateLayers()` - обновляет слои обработки новыми данными о распознанных картах
- `checkArea()` - выполняет шаблонное сравнение в указанной области экрана
- Возвращает объект MatchingData с количеством совпадений фона и карт

### 4. AdvisorEngine.java
**Назначение:** Интеллектуальная система рекомендаций для покерных решений

**Как работает:**
- Комбинирует несколько факторов принятия решения:
  * Equity factor - шансы выиграть банк
  * Sklansky factor - производительность по группам рук Скланиски
  * Stats factor - статистика противников (VPIP, PFR, агрессия)
  * Position factor - выгода от позиции за столом
  * Stage factor - этап турнира и размер стека
  * Dynamic factor - недавняя динамика действий за столом
- Использует адаптивное взвешивание - автоматически регулирует важность факторов в зависимости от доступных данных
- Применяет "мягкую" логику - постепенно меняет рекомендации по мере накопления уверенности
- Генерирует объяснимые рекомендации с указанием уверенности и обоснования

**Ключевые классы:**
- `Action` - перечисление возможных действий (FOLD, CHECK, CALL, RAISE, ALL_IN)
- `Recommendation` - содержит действие, уровень уверенности и объяснение

**Ключевой метод:**
- `recommend()` - главный метод, принимающий все параметры игры и возвращающий рекомендацию

**Как интегрируется с другими компонентами:**
- Получает данные о картах от Recognition через покерные калькуляторы
- Читает статистику противников из PlayerDatabase (обновляется из исторических данных экрана)
- Использует настройки из Settings (например, пороги агрессии, минимальное количество рук для статистики)
- Рекомендации отображаются в HUD через AdvisorChart, если включен соответствующий чекбокс

### 5. HoldemCalculator.java (и другие калькуляторы)
**Назначение:** Расчет шансов и вероятностей для покерных раздач

**Как работают:**
- Implement the Calculator interface с методом getWinRate()
- Используют либо точные комбинаторные расчеты, либо Монте-Карло симуляции
- Принимают известные карты (руку игрока и карты на доске)
- Возвращают вероятности выигрыша, проигрыша и ничьи против случайных рук оппонентов
- HoldemCalculator специфичен для Техасского Холдема
- Другие калькуляторы (Omaha, SixPlusHoldem, Sklansky) работают аналогично для своих вариантов покера

**Как используются AdvisorEngine:**
- AdvisorEngine вызывает соответствующий калькулятор basado на настройке игры (holdem/omaha/6plus)
- Получает массивы wins, losses, draws, iterations для каждого оппонента
- Извлекает equity (шансы выиграть) из этих данных для использования в рекомендациях

### 6. Настройки и чек박сы (из Settings.java и связанных файлов)
Хотя эти файлы не были явно запрошены для копирования, они важны для понимания работы чек박сов:

**Чекбоксы управления функциями:**
- `advisorMainEnabled` - включает/выключает основной HUD советника (пот, PIP, ник, блайнды, ставка)
- `advisorPlayerStatsEnabled` - включает/выключает HUD со статистикой игроков при клике на аватар

**Области мониторинга экрана:**
- `handArea` - прямоугольник, где искать карты игрока (handAreaX, handAreaY, handAreaW, handAreaH)
- `boardArea` - прямоугольник, где искать общие карты (boardAreaX, boardAreaY, boardAreaW, boardAreaH)

**Параметры советника, обновляемые из данных экрана:**
- `advisorVPIP` - процент добровольных взносов в банк (вычисляется из наблюдаемых действий)
- `advisorPFR` - процент повышений префлопа (вычисляется из наблюдаемых действий)
- `advisorAggressionFactor` - фактор агрессии (вычисляется из наблюдаемых действий)
- `advisorLast5Actions` - последовательность последних действий (F/C/R/A)

**Как работают чекбоксы:**
1. Пользователь переключает чекбокс в VisualizationFragment (SwitchMaterial)
2. Это обновляет соответствующее булево значение в Settings через SharedPreferences
3. При включении чекбокса:
   - advisorMainEnabled: показывает/скрывает элементы основного HUD советника
   - advisorPlayerStatsEnabled: показывает/скрывает панель со статистикой при клике на аватар
4. При выключении чекбокса соответствующие элементы HUD не отображаются
5. Настройки сохраняются и загружаются через SharedPreferences при запуске приложения

### Поток данных от экрана до рекомендации:

1. **Захват экрана:**
   - OverlayService.initScreenCapture() → создает виртуальный display
   - Виртуальный display непрерывно передает кадры в ChangeFinder
   - ChangeFinder обрабатывает кадры через RenderScript (фоновое вычитание, обнаружение изменений)

2. **Обработка изображений:**
   - ChangeFinder передает данные Recognition объектам
   - Recognition.java анализирует потенциальные карты (ранг, масть, уверенность)
   - Данные фильтруются временно для устранения ложных срабатываний
   - Геометрические данные (координаты, размеры) поддерживаются для пространственной привязки

3. **Покерные вычисления:**
   - Распознанные карты передаются в покерные калькуляторы (HoldemCalculator и т.д.)
   - Калькуляторы возвращают массивы wins/losses/draws/iterations vs оппонентов
   - Из этих данных извлекается equity (шансы выиграть)

4. **Профилирование игроков:**
   - PlayerDatabase отслеживает исторические действия оппонентов
   - Из этой истории вычисляются VPIP, PFR, агрессия и т.д.
   - Эти параметры обновляются по мере наблюдения за игрой

5. **Генерация рекомендации:**
   - AdvisorEngine.receive() получает:
     * Equity данные от калькуляторов
     * Sklansky данные (группы рук)
     * Статистику игроков из PlayerDatabase
     * Позицию за столом, размер стека, уровень блайндов
     * Последние действия за столом
   - Вычисляет факторы для каждого компонента решения
   - Применяет адаптивное взвешивание (автоматически регулирует важность факторов)
   - Комбинирует факторы в итоговую оценку
   - Преобразует оценку в конкретное покерное действие с объяснением

6. **Отображение HUD:**
   - Если advisorMainEnabled = true, AdvisorChart показывает рекомендацию
   - Отображает действие (FOLD/CHECK/CALL/RAISE/ALL_IN), уровень уверенности и объяснение
   - Обновляется в реальном времени по мере поступления новых данных с экрана

### Как это работает в другой среде:

Для переноса этой системы в другую среду вам понадобится:
1. Заменить платформозависимые части (Android-specific API) на аналоги в целевой среде
2. Сохранить основную логику:
   - Поток от захвата экрана → обработки изображений → распознавания карт → покерных расчетов → принятия решений
   - Адаптивное взвешивание факторов в AdvisorEngine
   - Persistence настроек через эквивалент SharedPreferences
   - Управление функциями через чекбокс-подобные переключатели
3. Адаптировать специфические для покера расчеты под ваш целевой язык/платформу
4. Сохранить структуру данных (карты, позиции, статистика игроков) для совместимости с логикой советника

Эти файлы содержат всю основную логику, необходимую для понимания и воспроизведения системы покерного HUD с экранным захватом, распознаванием карт и интеллектуальным советником.
