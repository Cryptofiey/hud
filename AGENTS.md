# ФУНДАМЕНТАЛЬНАЯ АРХИТЕКТУРА И ИГРОВЫЕ МОЗГИ POKER BOT

Этот документ является определяющей концепцией проекта и долгосрочным руководством для ИИ-моделей (включая любые будущие модели Google AI Studio). Изменять или удалять описанные здесь принципы **СТРОГО ЗАПРЕЩЕНО**. Все изменения логики и кода должны беспрекословно следовать этой архитектуре.

---

## УРОВЕНЬ 1 (L1) — Чистая математика GTO (Голое Мат. Ожидание)
*Статус: Настроен, оптимизирован.*

### Основная концепция L1:
* **Расчет математической истины**:
  * Расчет строится на голой математике по GTO без попыток予測 (прогнозировать) действия оппонентов на основе их психологии.
  * Вместо использования одного источника EV, L1 извлекает математическую правду из **4 или более независимых источников**, математически рассчитывает и складывает их результаты.
  * Благодаря извлечению из 4 источников минимизируется погрешность вычислений. Это позволяет оставаться в зоне чистых доказанных расчетов, исключая уход в зону догадок.
* **Главное условие стабильности**:
  * На уровне L1 **не должно быть сбоев в определении карт игрока и общих карт стола (Board)**. Полная точность, надежность и стабильность распознавания — фундамент всех последующих уровней.

---

## УРОВЕНЬ 2 (L2) — Индивидуальная калибровка по оппонентам
*Статус: В процессе разработки и исправления.*

### Архитектура сбора данных (Две зоны сбора информации):
Бот собирает, хранит и обрабатывает данные индивидуально по **КАЖДОМУ** игроку за столом, без пропусков, замалчиваний и "средних по палате" допущений. 

1. **Первая зона сбора (Исторические / Lifetime Профили)**:
   * Данные считываются только когда открыто окно профилей игроков на целевой платформе (где видны все 10 статистических параметров: VPIP, PFR, 3Bet, AF, WTSD, WSD и др.).
   * Работает по триггеру (пользователь или бот нажали кнопку сканирования профиля). Данные записываются в базу данных (БД). Игрок может переключаться между профилями оппонентов, пока бот поочередно не сохранит полную статистику каждого игрока.
   * Эти данные представляют собой **Lifetime (историческую) статистику**.

2. **Вторая зона сбора (Актуальный Игровой Стол)**:
   * Работает непрерывно прямо на зеленом холсте игрового стола.
   * Считывает все текущие игровые параметры за текущий турнир/сессию (актуальный VPIP, ставки, стеки).
   * Эти данные отражают **Текущую динамику/сессию**.

### Логика расчетов L2:
* **Индивидуальный обсчет каждого активного оппонента**:
  * Расчеты в L2 базируются на математике L1, но прогоняются через формулы корректировки под конкретных оппонентов с использованием параметров `VPIP`, `WSD`, `WTSD`.
  * Бот не выбирает какого-то одного "избранного" или среднего оппонента из списка. Он обрабатывает математическую модель поведения каждого активного оппонента в раздаче.
* **Гибкость и мягкая деградация (Fault Tolerance)**:
  * Если по какому-то игроку отсутствуют определенные данные, бот использует **дублирующие поля** из другого источника (например, сессионный VPIP со стола заменяет профильный VPIP, если тот не был отсканирован).
  * Если данных нет ни в одном источнику по конкретному игроку, система:
    * Применяет средние математические значения популяции, скорректированные на общие средние показатели со стола.
    * Применяет альтернативный механизм временного отключения учета персональных статов для этого игрока (или для всего стола).
  * Система сохраняет полную работоспособность при отказе любого количества вводных данных из обеих зон сбора. В самом крайнем случае алгоритм плавно "сжимается" до стандартных математических рамок EV (L1) с жесткими границами безопасности, полностью предотвращая нерациональные действия ("схождение с ума").
